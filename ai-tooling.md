# AI Tooling Usage

## Tools Used
- ChatGPT (OpenAI)
- Cursor IDE (AI-assisted coding)

---

## How AI Was Used

### 1. Code Generation
- Assisted in writing:
  - Flask routes
  - CI/CD pipeline (GitHub Actions)
  - Unit and integration tests

### 2. Debugging
- Helped resolve:
  - Import issues in pytest
  - CI/CD pipeline failures
  - GitHub push errors (large file handling)

### 3. System Design
- Guided:
  - CI/CD architecture
  - Deployment strategy (Render)
  - Testing pyramid (unit → integration → smoke test)

---

## What Worked Well

- Rapid iteration on CI/CD setup
- Faster debugging of environment issues
- Clear step-by-step guidance for deployment
- Structured approach to building a production-ready pipeline

---

## Limitations of AI

- Occasionally suggested incorrect assumptions about existing code
- Required manual verification for:
  - model behavior
  - data handling correctness
- Did not replace need for understanding ML fundamentals

---

## Reflection

AI significantly accelerated development but:
- Human validation was necessary
- Critical thinking was required to ensure correctness

The combination of AI assistance and manual verification resulted in a robust and deployable system.
